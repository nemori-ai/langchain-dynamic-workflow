# M3.5 · 多并行 run 可观测性 + quota 接线（fast-follow，可先于 M3）

> Status: committed plan (design_docs 批次). 出处: v0.3.0 调研收敛（见 `00-roadmap.md` 同名章节）——需求②"一个 session 多 workflow 并行" = NO-GAP，残项纯 ergonomics。本里程碑只补 host 侧聚合可见面 + 去掉一个 quota 静默忽略 footgun，**不碰并行机制本身、不碰跨进程持久化（= M3/D）**。

## 1. 目标

让 host 不必逐个 `run_id` 轮询就能看到自己**所有**在飞/已完成的 run，并让 `workflow_runs` state 通道在 run 落定时反映终态。

## 2. 现状（精读确认）

并行机制**已端到端跑通**，缺口仅在可观测面：
- `BgRunManager` 每 `(thread_id, run_id)` 一 `BgRunSlot`（`_background.py:226`），有 `active_run_count()`（`:240`）但**无枚举本 thread 全部 run 的方法**——slot 字典是私有的。
- 工具命令面只有 `run`/`run_script`/`status`/`resume`/`cancel`（`tool.py:61`）；`status` 单 run（`:282`）。无聚合命令。
- `workflow_runs` state 通道用 `operator.add` 纯追加（`middleware.py:49`）；记录在 launch 时写入 `status=RUNNING`（`tool.py:207-209`），**落定后从不刷新** → host 读到的永远是 RUNNING（stale）。
- `create_workflow_middleware(max_concurrent_runs=...)` 仅在**自建默认 manager** 时生效；传了 explicit `manager` 时**静默忽略**该参数（`middleware.py:181-199`）——no-silent-failure 红线下的 footgun。

## 3. 三块改动

### A — 聚合 `runs` 命令

- `_background.py`：新增 `@dataclass(frozen=True, slots=True) RunSnapshot{run_id, status: BgStatus, summary: str | None}` + `BgRunManager.list_runs(thread_id) -> list[RunSnapshot]`，枚举 `self._slots` 中该 thread 的 slot（终态带短 summary，在飞 summary=None）。只读快照，不漏可变 `BgRunSlot`。顺序：按 slot 插入序（dict 保序）。
- `tool.py`：`WorkflowToolSchema.command` Literal 加 `"runs"`；新增 `_runs_command(runtime)`（无需 run_id）调 `manager.list_runs(thread_id)`，对每条 join `run_specs.get(run_id)` 推 label（`name` spec → workflow 名；`script` spec → `_label_for_script`；缺失 → `"?"`），格式化为多行 `- {run_id} · {label} · {status}[ · {summary}]`；空列表回 `"runs: no runs on this thread yet."`。`workflow_tool` 分发加 `runs` 分支。工具描述 `_WORKFLOW_TOOL_DESCRIPTION` 增 `runs` 条目。

### B — `workflow_runs` 落定刷新

- `middleware.py`：新增 `_merge_workflow_runs(existing, incoming) -> list[dict]` reducer——**按 `run_id` upsert + 逐字段合并**：incoming 中与既有同 `run_id` 的记录就地合并字段（保 `workflow` label、覆盖 `status`），保留首见位置；新 `run_id` 追加。`WorkflowState.workflow_runs` 的 `Annotated[..., operator.add]` 换成 `Annotated[..., _merge_workflow_runs]`。
- `abefore_model` drain notices 后，除注入 `<workflow_notification>` 外，**同时**返回 `workflow_runs` 终态更新 `[{"run_id": n.run_id, "status": n.status.value} for n in notices]`，经 merge reducer 并入既有记录 → 通道反映终态。无 notices 时仍返回 `None`（不变）。

### C — quota 接线去歧义

- **偏离调研字面建议**（调研建议往 `create_workflow_tool` 加 `max_concurrent_runs`）：`create_workflow_tool` 不构造 manager（manager 是必填、且需与 middleware 共享以送达通知），加该参要么被忽略要么双源——都是 footgun。改为：`create_workflow_middleware` 在**同时**收到 explicit `manager` 且 `max_concurrent_runs is not None` 时**抛 `ValueError`**（消除静默忽略），docstring 明确 quota 归属 `BgRunManager`。default-manager 路径（manager=None）行为不变。

## 5. TDD 任务分解

- **A**：`test_list_runs_enumerates_thread_runs_with_status`（mixed in-flight/settled）；`test_list_runs_isolated_per_thread`；`test_runs_command_lists_runs_with_labels`；`test_runs_command_empty`。
- **B**：`test_merge_workflow_runs_upserts_by_run_id`（保 label 改 status、保位置、新 run 追加）；`test_abefore_model_emits_settled_status_update`（drain 后通道含终态）。
- **C**：`test_middleware_raises_on_manager_plus_quota_conflict`；`test_middleware_default_manager_honors_quota`（回归，不破坏既有）。
- 集成：host（中间件 + 工具）启 ≥2 并行 run → 一个落定 → `runs` 聚合视图反映 mixed 状态 + `workflow_runs` 通道终态刷新。

## 6. 验收门（遵 per-gap 方法论）

1. 完整 TDD，ruff + pyright strict 全绿；
2. 真模型 E2E（招牌：host 从道层 prompt 启 N 并行 run、查聚合 `runs` 并据各结果反应；遵 [[real-e2e-demo-must-exercise-headline-path]]，LangSmith tracing 关）；
3. user-facing 集成示例（host 接线、照抄即用）；
4. Codex 跨模型评审一轮；
5. evergreen 同步 + 独立重跑全门（遵 [[independently-verify-gate-claims]]）。

## 7. YAGNI 边界

- 不碰并行机制本身（已工作）。
- 不做跨进程/跨会话持久化（= M3/D）。
- 不做全局跨-run 预算池（roadmap 开放决策，待真实场景驱动）。
- `runs` 命令只列本 host thread 的 run（与现有 thread 隔离一致），不跨 thread 聚合。
