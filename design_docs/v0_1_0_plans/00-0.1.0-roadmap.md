# 0.1.0 分阶段实现路线图（Roadmap）

> **For agentic workers:** 本文件是 0.1.0 的**阶段总线**。每个阶段对应一份独立的 bite-sized TDD plan（`design_docs/v0_1_0_plans/0N-phase-N-*.md`），用 superpowers:subagent-driven-development 或 superpowers:executing-plans 逐任务执行。

**Goal:** 在 LangChain deepagents 上实现 Claude Code Dynamic Workflows 机制（控制流反转引擎），0.1.0 = 一个 host deepagent 能通过一个 workflow tool 后台跑编排脚本、只拿回最终结论。

**Architecture:** 见 [../01-engine-mechanism.md](../01-engine-mechanism.md)（引擎机制）与 [../02-architecture.md](../02-architecture.md)（对外形态）。骑 LangGraph `@entrypoint`/`@task` durable execution;自建 content-hash journal、确定性 guard、pipeline 调度、SandboxManager、async 后台 tool 机制;叶子 = deepagent。

**Tech Stack:** Python 3.12 · uv · langgraph 1.2.2 · langchain 1.3.2 · langchain-core 1.4.0 · deepagents 0.6.7 · pydantic 2 · pytest + pytest-asyncio · ruff · pyright(strict)。

---

## 全局工程约定（每阶段都遵守）

1. **TDD**：Red → Green → Refactor。每个任务一条失败测试 → 跑挂 → 最小实现 → 跑绿 → commit。
2. **绿色测试用 fake model，零真实 API key**：deepagent 的 e2e demo 在测试里用 `langchain_core.language_models.fake_chat_models.GenericFakeChatModel` / `FakeListChatModel`(或自定义返回脚本响应的 fake `BaseChatModel`)驱动,保证 `uv run pytest` **全绿、确定性、零成本**。纯单元层可直接用「fake `CompiledSubAgent` runnable」(返回 `{"messages":[AIMessage(...)]}`)绕开建 agent。真实模型版放 `examples/` 由 `LDW_DEMO_REAL_MODEL` env + API key 门控,**不进绿色测试路径**。
3. **每阶段交付三件套**：(a) 库代码 + 单测;(b) 一个 e2e 集成测试(fake model 驱动的 deepagent demo,**绿**);(c) `examples/0N_*.py` 可跑脚本。
4. **demo 连续演进**：后阶段可 refactor 前阶段的模块与 demo(demo 从 01 长到 06)。每次 refactor 后全套测试必须仍绿。
5. **每改一个模块就跑 `uv run ruff check . && uv run ruff format . && uv run pyright`**,零告警再继续(pyright strict)。
6. **测试输出重定向**：`uv run pytest -q > /tmp/ldw-test.log 2>&1; echo "EXIT=$?"; tail -30 /tmp/ldw-test.log`(见 `.claude/rules/testing.md`)。
7. **commit 频繁**:每任务一 commit;阶段收尾打里程碑 commit。
8. **import-linter 层边界**:Phase 6 引入 `[tool.importlinter]` `forbidden` contracts 守 L0/L1/L2;但从 Phase 1 起就按层组织模块,别跨层乱 import。

## 测试基础设施（`tests/` 共享脚手架，Phase 1 建立、各阶段复用）

- `tests/fixtures/fake_models.py`：`scripted_model(responses: list[str|AIMessage]) -> BaseChatModel`(确定性 fake)、`call_counter` 包装(统计 model 调用次数,用于验 journal cache-hit = 0 调用)。

## Phase 1 · 核心纵切片（里程碑 M1）

**Plan:** `01-phase-1-core-vertical-slice.md`

**里程碑 M1**：`run_workflow(script, roster)` 能跑一段调用 `await ctx.agent(...)` 的脚本、命中真实 deepagent 叶子、回填最终结论,且二次运行命中 journal 缓存(0 模型调用)。**同步执行**,不含 parallel/pipeline/sandbox/guard/budget。

**范围**：`_journal`、`_roster`、`_result`、`_leaf`、`_context`(仅 `agent()`)、`_engine`(@entrypoint + `run_workflow`)、测试脚手架。

**验收标准**：
- [ ] `run_workflow` 跑单 `agent()` 脚本返回回填后的字符串结论(fake-model deepagent)。
- [ ] 带 `schema=` 时返回校验过的对象(`ToolStrategy` 路径);无 schema 时返回末条非空 `AIMessage` 文本。
- [ ] 同 `thread_id` 二次运行:journal 命中 → **模型调用计数 = 0**(用 `call_counter` 断言)。
- [ ] journal key = `(prompt, agent_type, model, schema_json, isolation)` 的 sha256;改任一要素 → miss 重跑(测试覆盖)。
- [ ] journal **success-only**:叶子抛错 → 不写缓存(测试:第一次抛错、第二次重试仍 live)。
- [ ] `uv run ruff check . && uv run pyright` 零告警;`uv run pytest` 全绿。

**指标**：journal 单测 ≥6 个(key 稳定/各要素失效/success-only/序列化往返);集成测试 cache-hit 计数 == 0;pyright strict 0 error。

**e2e demo**：`examples/01_single_agent.py` —— 一个 deepagent(fake model)回答一个被脚本编排的问题。集成测试 `tests/integration/test_phase1_single_agent.py` 断言结论 + resume 缓存命中。

**依赖**：无(起点)。

## Phase 2 · 扇出：parallel + pipeline + 并发闸（里程碑 M2）

**Plan:** `02-phase-2-fanout.md`

**里程碑 M2**：脚本能 `await ctx.parallel([...])`(barrier)与 `await ctx.pipeline(items, *stages)`(无 barrier 流式)扇出多叶子,并发受显式上限约束,失败语义正确,journal 跨扇出与半途中断仍正确。

**范围**：`_concurrency`(Semaphore + `RunnableConfig.max_concurrency`)、`_pipeline`(bounded `asyncio.Queue` 调度)、`_context` 加 `parallel`/`pipeline`;refactor `_engine` 容纳调度器。

**验收标准**：
- [ ] `parallel([thunk,...])` 返回**按序** list;thunk 抛错 → 该位 `null`,**调用本身不 raise**;用例验 `.filter` 后可用。
- [ ] `pipeline(items, *stages)`:item 各自独立穿越(无 barrier,验 A 在 stage3 时 B 仍 stage1);stage 抛错 → 该 item `null` 跳后续;结果**按输入下标保序**。
- [ ] 并发上限:instrumented fake 断言**最大在飞 ≤ min(16,cores-2)**(或显式配置值)。
- [ ] journal:parallel/pipeline 内每叶子各自 content-hash 缓存;**半途中断后 resume**:已完成叶子命中缓存、未完成 live(集成测试)。
- [ ] ruff/pyright/pytest 全绿。

**指标**：并发上限断言测试;失败语义(parallel null / pipeline drop)各 ≥1;无-barrier 交错测试;半途 resume 测试;pipeline 异常不死锁测试。

**e2e demo**：`examples/02_fanout.py` —— 扇出 N 个研究叶子(parallel)+ 一个 2-stage pipeline。集成测试绿。

**依赖**：Phase 1。

## Phase 3 · 确定性 guard + budget + 进度（里程碑 M3）

**Plan:** `03-phase-3-determinism-budget.md`

**里程碑 M3**：journal-divergence backstop 在重放分叉时 fail-loud;`budget` 共享池可强制、且 `spent()` 重放可重建;`phase()`/`log()` 出进度且重放幂等。

**范围**：`_determinism`(divergence backstop;AST gate 占位接口,完整 AST gate 在 L2/Phase 5 skill 路径前不强求)、`_budget`(共享池 + usage 入 journal 重建 + `UsageMetadataCallbackHandler` 转发)、`_context` 加 `phase`/`log`/`budget`。

**验收标准**：
- [ ] 重放时若脚本产出的 `agent()` 调用序列与 journal 记录不匹配 → 抛 `WorkflowDeterminismError`(清晰信息),**不喂错位缓存**。
- [ ] `budget.spent()` 在 resume 时由 journal 中每叶子 usage **重建出与首次相同的累积值**(断言)。
- [ ] `spent() >= total` 后新 `agent()` 抛 `WorkflowBudgetExceededError`;在飞叶子完成并保留结果。
- [ ] `budget.total` 守卫:无 total 时 `remaining()` 为 inf(loop-until-budget 范式可用)。
- [ ] `phase`/`log` 输出可捕获;重放不重复投递(幂等)。
- [ ] ruff/pyright/pytest 全绿。

**指标**：divergence 检出测试;budget 重放确定性测试;budget-cap 强制测试;phase/log 幂等测试。

**e2e demo**：`examples/03_loop_until_budget.py` —— loop-until-budget 累积式扇出。集成测试绿。

**依赖**：Phase 1、2。

## Phase 4 · Sandbox：per-leaf 隔离与生命周期（里程碑 M4）

**Plan:** `04-phase-4-sandbox.md`

**里程碑 M4**：`needs_execution` 叶子获得隔离 backend(身份从 journal key 派生);并行执行叶子互不串味;纯推理叶子不分配 sandbox;SandboxManager 自管 TTL/池/配额/stop。

**范围**：`_sandbox`(SandboxManager + per-leaf backend 实例构造,**弃 deprecated factory**)、`_leaf` 接入分层准入与 sandbox、CompositeBackend `/shared/` hand-off(路径规范化)。

**验收标准**：
- [ ] `needs_execution=True` 叶子拿到隔离 backend;两个并行执行叶子写同名文件**互不可见**(隔离测试,用 local/state backend)。
- [ ] sandbox 身份从 journal key 派生:**retry / resume 时稳定**(同叶子 → 同身份 → 同 backend)。
- [ ] `needs_execution=False` 叶子走 StateBackend、**不分配 sandbox**(断言 manager 未 acquire)。
- [ ] SandboxManager:idle TTL 回收;池耗尽 → 排队(背压);`stop()` 被调用清理。
- [ ] `/shared/` hand-off:producer 写、consumer 读;路由前路径规范化防 `../` 穿越(测试)。
- [ ] ruff/pyright/pytest 全绿。

**指标**：隔离测试;身份稳定测试;分层准入测试;TTL/配额测试;路径穿越防护测试。

**e2e demo**：`examples/04_sandbox_artifacts.py` —— 执行叶子在隔离 sandbox 写/读产物 + `/shared/` 交接。集成测试绿(local/state backend)。

**依赖**：Phase 1、2(、3)。

## Phase 5 · 对外形态：tool + middleware + async 后台 + skills（里程碑 M5）

**Plan:** `05-phase-5-outward-form.md`

**里程碑 M5**：一个 host deepagent(fake model 脚本化)通过 `workflow_tool(run, script)` **后台**启动一次 workflow、立即拿 run_id 占位、完成后经 `abefore_model` 注入通知、再 `status`/`get_result` 取最终结论;skills 注入 host prompt;`workflow()` 一级嵌套。

**范围**：`tool.py`(多命令 run/status/resume/cancel)、`_background`(BgRunManager/BgRunSlot/ResultStore-memory)、`middleware.py`(贡献 tool via `.tools` + `abefore_model` 注入 + `async_runs` state channel)、`_context` 加 `workflow()`、`skills/`(SKILL.md 编排教学集)。

**验收标准**：
- [ ] host deepagent 调 `workflow_tool(run, script=...)` → **即返** run_id 占位 ToolMessage(不阻塞 host 回合)。
- [ ] 后台 `asyncio.create_task` 跑 `run_workflow`;done callback 入队;**下一轮 `abefore_model` 注入 `<workflow_notification>`**(集成测试断言注入)。
- [ ] `status`/`get_result(run_id)` 返回结果(大结果 offload 到 ResultStore + 摘要/句柄)。
- [ ] `resume`/`cancel` 命令生效。
- [ ] skills 经 `create_deep_agent(skills=[...])` 进 host prompt(断言 skill 元数据出现)。
- [ ] `workflow()` 一级嵌套;二级嵌套抛错。
- [ ] ruff/pyright/pytest 全绿。

**指标**：e2e「host agent 驱动后台 workflow」测试绿;通知注入测试;status/resume/cancel 测试;skills 加载测试;嵌套守卫测试。

**e2e demo**：`examples/05_host_agent_workflow.py` —— **旗舰 demo**:host deepagent 后台跑 workflow 并取回结论。集成测试绿。

**依赖**：Phase 1–3(、4)。

## Phase 6 · 0.1.0 收口：集成 / 硬化 / 可观测 / 文档（里程碑 M6 = 0.1.0）

**里程碑 M6（0.1.0）**：一个贴近真实的多阶段 capstone workflow(parallel + pipeline + budget + sandbox + 对抗式验证),由 host deepagent 后台驱动,**端到端全绿**;层边界机械守护;公共 API 文档化;打 0.1.0。

**范围**：import-linter contracts、可观测性 hook(phase/log → 可选 trace;usage rollup)、capstone demo、覆盖率门、README/quickstart、CHANGELOG、version bump。

**验收标准**：
- [ ] capstone demo 端到端绿(fake model);真实模型变体文档化(env 门控)。
- [ ] `import-linter` 通过(L0/L1/L2 `forbidden` contracts);`uv run ruff check .`、`uv run pyright` 零告警;`uv run pytest` 全绿。
- [ ] 核心模块覆盖率 ≥ 目标(建议 line ≥85%;`agent/parallel/pipeline/journal/resume/background` 必须有集成测试)。
- [ ] 公共 API 稳定 + README quickstart 可照着跑通;CHANGELOG;`version = "0.1.0"`。

**指标**：capstone e2e 绿;import-linter 绿;覆盖率%;pyright 0;ruff 0。

**e2e demo**：`examples/06_capstone.py` —— 全特性 workflow(由 host deepagent 驱动)。

**依赖**：Phase 1–5。
