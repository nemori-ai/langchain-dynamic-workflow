## Python 代码撰写通用规则

### 作用域

本规则适用于所有的 .py 脚本 以及 .md 文件的 python 代码块。

### 规则

#### 代码风格

撰写 python 代码时，总是默认遵循**[谷歌风格指南（Google Python Style Guide）](https://google.github.io/styleguide/pyguide.html)**进行撰写；
然后，当撰写的是 .py 脚本时，使用 `uv run` 执行 ruff 和 pyright 进行检查，并根据检查结果进行优化以确保代码质量。

#### Python 版本

强制使用 **py312** 版本；
在不影响代码可读性的前提下，充分使用本版本下推荐的、能提高代码性能的语法和语法糖机制。

#### 面向 AI 友好的编程实践

- 类、函数、变量等的命名需足够直观和语义化，避免过度简写和缩写
- docstring 和 comment 充分，对复杂逻辑需用通俗易懂的语言说明算法流程
- 异常信息描述要求详细
- **总是**使用 type hints 或 annotations 来增强类型含义的可读性，适当使用范型

#### 变量命名补充

- 总是使用 python 标准风格的规范命名方式
- 单例在模块内直接实例化的话，单例对象用私有变量命名原则，单例实例用常量命名原则

#### 元编程和并发编程应用

- 你精通 python 的各种元编程机制，总是能够在适当的时机使用元编程技术来提高代码块的利用率和程序性能
- 你精通 python 的多线程和多进程原理
- 你精通 python 的协程原理和异步实现机制

#### 包和模块导入
- 总是按照 python 内置库 -> 安装的第三方库 -> 项目内的包和模块 的顺序进行导入
- 模块导入顺序按照前缀字符的 ASCII 码顺序从小到大排列
- 内部包和模块导入时：三层目录距离之内(`from ...xxx import yyy`)使用相对导入，三层目录距离之外使用绝对导入
- `__init__.py` 中只短路外部可能使用到的模块和对象，不要事无巨细地都写入到其中
- 除非满足例外条件，否则**永远**不要在函数闭包内执行模块导入，**总是在模块顶层进行导入**
  - 例外：可能引发递归导入的时候

#### 枚举的使用

- 总是使用多继承，前者确定值类型，例如：`class Color(str, Enum)`
- 枚举名采用常量命名原则，枚举值采用变量命名原则，例如：`BLACK = "black"`
